# 🚀 Vercel 網頁部署 - 最簡單的方式

## 5 分鐘完成部署！

### 步驟 1: 前往 Vercel（1 分鐘）

1. 開啟瀏覽器，前往：**https://vercel.com/new**
2. 點擊 **「Continue with GitHub」** 或 **「Sign Up」**
3. 使用您的 GitHub 帳號登入（`RheC1e`）

### 步驟 2: 匯入專案（1 分鐘）

1. 登入後，您會看到「Import Git Repository」頁面
2. 在搜尋框中輸入：`rhema-pwa-demo`
3. 或直接選擇：**`RheC1e/rhema-pwa-demo`**
4. 點擊 **「Import」**

### 步驟 3: 設定專案（1 分鐘）

Vercel 會自動偵測設定，您只需要確認：

- ✅ **Framework Preset:** `Vite`（已自動偵測）
- ✅ **Build Command:** `npm run build`（已自動設定）
- ✅ **Output Directory:** `dist`（已自動設定）
- ✅ **Install Command:** `npm install`（已自動設定）

**不需要修改任何設定，直接點擊「Deploy」！**

### 步驟 4: 等待部署（2 分鐘）

1. Vercel 會開始建置專案
2. 您可以看到即時的建置日誌
3. 建置完成後，會顯示「Congratulations!」
4. 點擊提供的網址即可訪問您的網站！

## 🎉 完成！

部署完成後，您會得到：
- **生產環境網址：** `rhema-pwa-demo.vercel.app`（或類似）
- **自動 HTTPS：** 已自動設定
- **全球 CDN：** 自動啟用

## 🔄 自動部署已啟用

之後每次您推送程式碼到 GitHub：
1. Vercel 會自動偵測變更
2. 自動執行建置
3. 自動部署到生產環境

**完全自動化！** 🚀

## 📝 後續設定（可選）

### 自訂網域

1. 前往 Vercel Dashboard → 您的專案 → Settings → Domains
2. 新增您的網域（例如：`rhema.com.tw`）
3. 按照指示設定 DNS 記錄

### 環境變數（如果需要）

1. 前往 Vercel Dashboard → 您的專案 → Settings → Environment Variables
2. 新增需要的環境變數

## 🆘 遇到問題？

### 建置失敗

1. 在 Vercel Dashboard 查看建置日誌
2. 檢查錯誤訊息
3. 確認 `package.json` 中的依賴都正確

### 無法找到倉庫

1. 確認已使用正確的 GitHub 帳號登入
2. 確認倉庫名稱是 `rhema-pwa-demo`
3. 確認倉庫是公開的，或您有權限訪問
